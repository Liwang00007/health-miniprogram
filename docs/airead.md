# [FitPlan 健身助手小程序]需求文档

---

## 一、小程序核心用途 / 产品定位

FitPlan 是一款面向健身爱好者的**智能训练与饮食管理小程序**，帮助用户记录运动与饮食数据、制定科学训练计划，并通过 AI 积分系统提供个性化训练方案与饮食建议，实现"记录—规划—执行—优化"的完整健身闭环。

**核心功能：**

- **训练记录**：记录每次训练的动作、组数、重量、时长，支持历史查询与数据可视化
- **饮食记录**：记录每日三餐及加餐的食物品类、热量、蛋白质等营养数据
- **训练目标**：设定并追踪阶段性健身目标（增肌、减脂、塑形等），支持目标拆解与进度跟踪
- **AI 训练计划**：消耗积分由 AI 生成个性化力量训练计划，自动保存至目标或教程模块
- **AI 饮食计划**：消耗积分由 AI 生成个性化饮食方案，适配用户的训练目标与热量需求
- **教程库**：提供动作教学视频/图文、训练知识科普，用户可收藏与查看 AI 生成的计划

## 二、主要功能详情

### 2.1 全局基础布局

- **底部导航栏（TabBar）**：固定四个 Tab ——「记录」「目标」「教程」「我的」
  - 「记录」Tab：训练记录 + 饮食记录的聚合入口，默认展示当日记录概览
  - 「目标」Tab：训练目标列表 + 训练计划 + 饮食计划，统一管理
  - 「教程」Tab：动作教程库 + AI 生成的计划存档
  - 「我的」Tab：个人中心，含积分、设置、数据统计等
- **全局顶部导航**：各页面独立标题栏，右侧根据页面场景放置"添加记录""筛选""搜索"等功能按钮
- **全局悬浮按钮**：记录页右下角固定"+"悬浮按钮，点击展开"添加训练记录 / 添加饮食记录"快捷入口
- **登录/注册**：首次进入强制跳转登录页（微信一键授权登录），未登录状态下仅可浏览教程库公开内容

### 2.2 首页功能（记录 Tab）

**页面内容：**

- **今日概览卡片**：展示今日已消耗/摄入热量环形图、训练时长、目标完成进度百分比
- **训练记录列表**：按时间倒序展示当天训练记录条目（动作名称 + 组数 × 重量 + 时长），支持左滑删除
- **饮食记录列表**：按早/中/晚/加餐分组展示当天饮食记录（食物名称 + 热量），支持左滑删除
- **历史记录入口**：页面顶部日期切换器，支持选择日期查看历史记录，默认展示当天

**按钮与交互：**

- 点击概览卡片 → 跳转至该日的详细数据统计页
- 点击右上角"日历"图标 → 弹出日历组件，选择日期跳转至对应日记录
- 点击右下角"+"悬浮按钮 → 弹出底部弹窗，选择「训练记录」或「饮食记录」进入添加页面
- 每条训练记录点击 → 进入训练详情页（可编辑）
- 每条饮食记录点击 → 进入饮食详情页（可编辑）

**跳转逻辑：**

- 「添加训练记录」→ 训练记录编辑页 → 保存后返回记录页并刷新列表
- 「添加饮食记录」→ 饮食记录编辑页 → 保存后返回记录页并刷新列表
- 无网络时页面缓存上次数据，顶部显示"网络异常，数据可能不是最新"提示条

### 2.3 我的页面功能

**未登录状态：**

- 显示默认灰色头像 + "点击登录"文案
- 点击头像区域 → 跳转微信授权登录页
- 登录成功后自动刷新页面数据，头像与昵称同步微信信息
- 未登录不显示积分、统计数据；教程库仅展示公开内容

**已登录状态：**

- **个人信息区**：头像（微信头像）、昵称、积分余额（醒目展示，点击可查看积分明细）
- **训练统计区**：累计训练次数、累计训练时长、连续打卡天数，点击进入详细统计页
- **功能入口列表**：
  - 我的训练计划 → 跳转目标页筛选"AI 计划"分类
  - 我的饮食计划 → 跳转目标页筛选"饮食计划"分类
  - 我的收藏 → 收藏的教程与计划列表
  - 数据导出 → 支持导出训练/饮食 CSV 数据
  - 提醒设置 → 训练提醒、饮食记录提醒开关
  - 设置 → 隐私政策、用户协议、清除缓存、退出登录
- **底部按钮**：「退出登录」，二次确认后清除本地数据并跳转登录页

### 2.4 核心操作功能（训练记录 / 饮食记录 / AI 制定计划）

#### 2.4.1 训练记录

**添加训练记录步骤：**

1. 点击记录页右下角"+" → 选择「添加训练记录」
2. 进入训练记录编辑页，依次填写：
   - 训练日期（默认当天，可修改）
   - 训练类型（下拉选择：力量训练 / 有氧 / HIIT / 拉伸 / 自定义）
   - 选择动作（从动作库搜索/选择，支持添加自定义动作）
   - 每组记录：组数序号、重量(kg)、次数、休息间隔
   - 训练总时长（手动输入或计时器记录）
   - 备注（可选）
3. 点击「保存」→ 数据写入，返回记录页并刷新列表

**按钮交互逻辑：**

- 「添加动作」按钮 → 弹出动作搜索/选择面板，支持按部位（胸/背/腿/肩/臂/核心）筛选
- 「添加组」按钮 → 在当前动作下新增一组记录行
- 「删除动作」→ 长按或点击删除图标，二次确认后移除
- 「计时器」按钮 → 弹窗内置倒计时/正计时器，结束自动填入时长字段
- 「保存」→ 表单校验通过后提交，缺少必填项时对应字段标红提示

#### 2.4.2 饮食记录

**添加饮食记录步骤：**

1. 点击记录页右下角"+" → 选择「添加饮食记录」
2. 进入饮食记录编辑页，依次填写：
   - 记录日期（默认当天）
   - 餐别（早/中/晚/加餐）
   - 食物名称（支持搜索常见食物库，含热量/蛋白质/碳水/脂肪预设数据）
   - 分量（克/g）
   - 自动计算摄入热量与营养素
   - 备注（可选）
3. 点击「保存」→ 数据写入，返回记录页并刷新列表

**按钮交互逻辑：**

- 「搜索食物」输入框 → 实时模糊匹配食物库，展示食物名称及每 100g 营养数据
- 「自定义食物」→ 食物库无匹配时，手动输入名称及营养数据
- 「拍照识别」→ 调用 OCR/视觉识别（预留能力），自动识别食物并填入，消耗 1 积分
- 「保存」→ 校验通过后提交

#### 2.4.3 AI 制定训练 / 饮食计划

**操作步骤：**

1. 在「目标」Tab 或「记录」页顶部入口点击「AI 制定计划」
2. 进入 AI 计划定制页，选择计划类型：
   - AI 力量训练计划（未来 N 天）
   - AI 饮食计划（未来 N 天）
3. 填写/选择定制参数：
   - 训练计划：目标肌群、训练天数（1-14 天）、训练水平（新手/进阶/高阶）、可用器械、是否有伤病史
   - 饮食计划：目标（增肌/减脂/维持）、每日目标热量、饮食偏好（正常/高蛋白/低碳/生酮等）、过敏食材
4. 点击「生成计划」→ 弹出积分消耗确认弹窗（消耗 X 积分）
5. 确认后调用 AI 接口生成计划，展示加载动画（预计等待 5-15 秒）
6. 生成完成后进入计划预览页，展示每日训练动作安排 / 每日饮食菜单

**按钮交互与跳转规则：**

- 「生成计划」→ 积分不足时弹出提示"积分不足，无法生成"，引导至积分获取页面
- 「确认消耗积分」弹窗 → 显示当前积分余额和消耗数量，点击「确认」发起请求
- 「保存计划」→ 训练计划自动保存到「目标」→「训练计划」列表；饮食计划保存到「目标」→「饮食计划」列表
- 「保存到教程」→ 同时复制一份保存到「教程」Tab，方便随时查阅
- 计划预览页支持「重新生成」（再次消耗积分）和「编辑调整」（手动微调，不消耗积分）
- AI 生成失败时展示友好提示"生成失败，积分已退还"，支持重试

**积分规则：**

- 新用户注册赠送 10 积分
- 每次 AI 生成训练计划消耗 3 积分
- 每次 AI 生成饮食计划消耗 2 积分
- 积分获取方式：每日签到 +1 积分、连续打卡 7 天 +3 积分、完成训练目标 +2 积分

## 三、界面与使用体验要求

### 3.1 界面样式规范

- **配色方案**：主色调采用深灰 + 活力橙（#FF6B35），强调健身的力量感与活力；背景色浅灰（#F5F5F5），卡片白色圆角
- **字体规范**：标题使用系统默认加粗字体，正文使用系统默认常规字体；数字数据使用等宽数字字体保证对齐
- **图标风格**：统一使用线性图标，底部导航选中态为主色填充，非选中态为灰色线性
- **卡片设计**：概览数据卡片使用圆角阴影，列表项卡片之间间距 12px，卡片内边距 16px
- **空状态**：无数据页面展示插画 + 引导文案 + 操作按钮，避免空白页
- **深色模式**：暂不支持，后续版本视用户反馈添加

### 3.2 交互使用体验

- **操作反馈**：所有按钮点击有 200ms 缩放动效；数据提交后展示 Toast 提示（成功/失败）；删除操作需二次确认
- **加载状态**：列表加载使用骨架屏，AI 生成计划使用进度条 + 趣味文案轮播（"正在分析你的训练需求…""正在匹配最佳动作组合…"）
- **下拉刷新**：记录页、目标页、教程页均支持下拉刷新
- **手势操作**：列表项支持左滑删除；训练记录详情页支持左右滑动切换日期
- **异常处理**：网络异常展示断网提示及重试按钮；AI 接口超时（>30s）自动取消并提示
- **键盘适配**：输入框聚焦时页面自动上移，避免被键盘遮挡

### 3.3 性能速度要求

- **首屏加载**：小程序冷启动 ≤ 2 秒，热启动 ≤ 1 秒
- **页面切换**：Tab 切换无明显白屏，页面渲染 ≤ 500ms
- **数据提交**：记录保存接口响应 ≤ 1 秒，提交后立即更新本地数据（乐观更新）
- **AI 生成**：首次生成 ≤ 15 秒，超时自动中断并退还积分
- **图片加载**：教程图片使用懒加载 + WebP 格式，支持渐进式加载
- **离线能力**：训练/饮食记录支持离线添加，网络恢复后自动同步

### 3.4 安全相关要求

- **用户认证**：仅支持微信授权登录，Token 有效期 7 天，过期自动刷新
- **数据传输**：全站 HTTPS，敏感数据（用户个人信息、训练数据）传输加密
- **数据存储**：用户数据存储于云数据库，按用户 ID 隔离，禁止跨用户访问
- **隐私合规**：用户数据收集遵循《个人信息保护法》，隐私政策在登录前强制展示
- **AI 内容安全**：AI 生成内容经过安全过滤，避免不当饮食建议或危险训练方案
- **积分防刷**：签到、打卡积分获取有频次限制，防止脚本刷积分

## 四、核心功能优化建议

### 4.1 优化目标

围绕**降低记录门槛、提升计划粘性、增强数据价值**三个方向，使小程序从"记录工具"升级为"智能健身伴侣"，提升用户留存与日活。

### 4.2 具体优化方案

#### 优化一：语音快捷记录（降低记录门槛）

- **问题**：训练过程中手动输入动作、重量、次数体验较差，用户容易放弃记录
- **方案**：在训练记录页增加"语音记录模式"，用户口述如"杠铃卧推 60 公斤 8 次 3 组"即可自动解析并填入表单；语音识别结果可二次编辑确认
- **预期效果**：单次训练记录时间从 2-3 分钟缩短至 30 秒以内

#### 优化二：训练计划执行追踪（提升计划粘性）

- **问题**：AI 生成计划后用户可能不执行，计划模块与记录模块割裂
- **方案**：在计划详情页增加"开始执行"按钮，点击后该计划进入"进行中"状态，每日自动创建当日训练/饮食条目待办提醒；用户在记录页完成当天计划内容时自动标记完成，进度实时回显到计划页
- **预期效果**：计划执行率从预估 30% 提升至 60% 以上

#### 优化三：数据可视化与周报推送（增强数据价值）

- **问题**：用户记录了大量数据但缺乏直观反馈，无法感知进步
- **方案**：在"我的"页面增加「训练周报」入口，以图表形式展示本周训练容量趋势、肌群训练分布雷达图、热量摄入 vs 消耗对比；每周一自动推送订阅消息提醒查看周报
- **预期效果**：用户周活跃天数从平均 2-3 天提升至 4-5 天

#### 优化四：社交激励与积分生态（提升用户增长）

- **问题**：积分获取渠道有限，用户生成 AI 计划的频次受限
- **方案**：增加"邀请好友得积分"（每邀请 1 人注册双方各得 3 积分）、"训练打卡分享海报"功能；好友间可互看训练动态（可选隐私开关）
- **预期效果**：通过社交裂变降低获客成本，积分消耗提升带动 AI 功能使用频次增长

#### 优化五：智能推荐与自适应计划（AI 能力升级）

- **问题**：当前 AI 计划为一次性生成，不会根据实际执行情况调整
- **方案**：在用户完成一个训练周期后，AI 根据实际完成数据（实际重量/次数 vs 计划目标）自动推荐下一周期计划调整建议（如增重、增加组数），用户确认后消耗积分生成优化版计划
- **预期效果**：形成"执行—反馈—优化—再执行"的 AI 闭环，提升长期留存
